Cross-Chain Payments

ThronosBuild accepts payments across multiple networks. Fees collected on external chains are used to mint wrapped THR tokens on those networks while locking native THR in our liquidity pools — creating real cross-chain liquidity for the Sentinel trading pools.

Cross-chain fee split: 50% treasury, 25% burn, 25% LP pools.
All external quotes are generated server-side with minimum floor prices to prevent underpricing.

Documentation

Everything you need to build mobile apps with ThronosBuild.

Getting Started

  1. Connect your wallet
  2. Click "New Build" and choose your source type (GitHub, GitLab, or ZIP Upload)
  3. Select platform (Android/iOS/Both) and build type
  4. Confirm payment in THR tokens
  5. Watch your build in real-time via WebSocket logs
  6. Download your APK/IPA when the build completes

GitHub & GitLab Builds

  • GitHub — Public repos work without credentials. Private repos require a Personal Access Token in the URL.
  • GitLab — Same pattern; use a GitLab Project Access Token.
  • Branch defaults to main; change it to any valid ref.

ZIP Upload Builds

  • Select ZIP Upload as the source type in the New Build modal.
  • Choose a Project Type or leave it on Auto-detect.
  • Optionally specify a Project Path if your app lives in a subdirectory.
  • Source ZIP is deleted after build completion.
  • Final build artifacts (APK/AAB/IPA) are retained for 7 days.

Supported Frameworks

  • Flutter — Dart/Flutter apps (pubspec.yaml auto-detected)
  • React Native — Standard RN projects
  • Expo — Managed and bare Expo workflows
  • Capacitor — Ionic/Capacitor hybrid apps
  • Gradle (Native Android) — Any standard Gradle-based Android project
  • Unity — Unity Android APK/AAB export (Android build target required)

iOS note: iOS (IPA) builds require a valid Apple Distribution certificate and provisioning profile. Ad-hoc and App Store distribution are both supported when iOS builds are enabled.

Artifact Retention

  • Build artifacts (APK, AAB, IPA) are kept for 7 days after a successful build.
  • Source ZIPs are deleted immediately after build completion.
  • Download your artifacts from the build detail page before they expire.

API Reference

POST /api/v1/builds — Submit new build
GET /api/v1/builds/:id — Get build status
GET /api/v1/builds/:id/logs — Get build logs
POST /api/v1/builds/:id/cancel — Cancel build
GET /api/v1/builds/:id/download/:platform — Download artifact
POST /api/v1/builds/preflight — Get validated quote (required for cross-chain)
POST /api/v1/uploads/project-zip — Upload ZIP source
GET /api/v1/status/pricing — Get THR prices + external floors
GET /api/v1/status/stats — Get system stats
